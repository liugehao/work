字段设计¶

  • 尽可能使用更小的数据类型，如 TINYINT、smallint,MEDIUMINT、INT、BIGINT (如int(11)的11代表客户端显示宽度，并不是取值范围,tinyint -2^8-2^8-1,smallint -2^15-2^15-1 int -2^31-2^31-1 bigint -2^63-2^63-1)
  • 尽量少用 TEXT、BLOB 等专有类型 (用链接代替)
  • 字符型，数值型字段类型不能混合使用，依赖后期转换
  • 相同字段不同表中的类型和长度要一致
  • 字段名称不能使用关键字
  • 不要指定字段级编码，建议全库统一
  • 默认值要规范，例如日期不要使用 0000-00-00
  • 不要用自增ID做主键
  • 不要使用外键
  • 事务相关记录保留时间戳，建议只增不改；在必须对记录进行修改的时候，保留更改时间戳
